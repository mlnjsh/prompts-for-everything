# Presentation Prompts

> Prompts for creating presentations, slide decks, pitch decks, and visual storytelling.

**Best with:** Gamma, ChatGPT, Claude, Beautiful.ai, Canva AI

---

## Slide Deck Structure

### 1. Conference Talk
```
Create a presentation outline for a [X-minute] conference talk:
Topic: "[talk title]"
Audience: [describe — technical level, interests]
Key takeaway: [the ONE thing they should remember]

Structure:
1. Title slide (talk title, your name, event name)
2. Hook slide (question, stat, or story that grabs attention)
3. Agenda/roadmap (3-4 main sections)
4. Section 1: [topic] — 3-4 slides with key points
5. Section 2: [topic] — 3-4 slides with key points
6. Section 3: [topic] — 3-4 slides with key points
7. Demo/example slide
8. Key takeaways (3 bullet points max)
9. Q&A slide
10. Thank you + contact info

For each slide: title, 2-3 bullet points, suggested visual, speaker notes (what to say).
Total slides: 15-20.
```

### 2. Investor Pitch Deck
```
Create a 12-slide investor pitch deck for [startup]:

Problem: [the problem]
Solution: [your solution]
Market: [target market and size]
Traction: [metrics if any]
Ask: [$X funding for Y purpose]

Slides:
1. Cover (company name, tagline, logo description)
2. Problem (make them feel the pain)
3. Solution (clear and simple)
4. Demo/Product (screenshots or flow)
5. Market Size (TAM/SAM/SOM with sources)
6. Business Model (how you make money)
7. Traction (metrics, growth chart)
8. Competition (2x2 matrix positioning)
9. Team (key members and superpowers)
10. Go-to-Market (first 12 months)
11. Financials (3-year projection highlights)
12. The Ask (amount, use of funds, timeline)

For each slide: title, key message, data points, visual description.
```

### 3. Quarterly Business Review
```
Create a QBR presentation for [team/department]:

Quarter: [Q1/Q2/Q3/Q4 YYYY]
Audience: [leadership, board, team]
Key metrics: [list 5-7 KPIs]

Slides:
1. Title + quarter highlights (3 wins in one line each)
2. Executive summary (5 sentences max)
3. KPI dashboard (table or scorecard format)
4. Revenue/performance deep dive (charts described)
5. What went well (top 3 wins with impact)
6. What didn't go well (top 3 misses with learnings)
7. Customer/market insights
8. Competitive landscape update
9. Next quarter priorities (3-5 initiatives)
10. Resource asks (what you need)
11. Appendix (supporting data)

For each slide: content and suggested chart types.
```

### 4. Training/Workshop
```
Create a training presentation on [topic]:

Duration: [X hours]
Audience: [experience level]
Format: [lecture / workshop / hybrid]

Structure:
1. Welcome + agenda + learning objectives
2. Why this matters (motivation, context)
3-6. Core content slides (concept → example → practice for each topic)
7. Hands-on exercise (instructions slide)
8. Common mistakes and how to avoid them
9. Quick quiz / knowledge check (5 questions)
10. Resources for further learning
11. Q&A
12. Feedback form link

Include: discussion prompts, breakout activities, and timing for each section.
```

## Slide Design Tips

### 5. Slide Makeover
```
I have a slide with this content:
[paste your current slide text — usually too much text]

Redesign it following these principles:
1. One idea per slide (split if needed)
2. Maximum 6 words per bullet point
3. Suggest a visual that replaces text (diagram, icon, photo)
4. Write speaker notes (what to SAY vs what's ON the slide)
5. Suggest the layout (full image, split, centered, etc.)

Rule: If you can say it, don't write it on the slide.
```

### 6. Data Slide Designer
```
I need to present this data on a slide:
[paste your data — numbers, percentages, comparisons]

Key message: [what should the audience conclude]

Design the slide:
1. Choose the right chart type and justify why
2. Title that states the insight (not just the topic)
   Bad: "Q4 Revenue"
   Good: "Q4 Revenue Grew 34%, Exceeding Target by 12%"
3. Highlight the key data point (color, size, annotation)
4. Remove chart junk (unnecessary gridlines, decorations)
5. Add a one-sentence callout box with the takeaway
6. Suggest colors (max 3, with one highlight color)
```

### 7. Story Arc Presentation
```
Transform this information into a story-driven presentation:

Topic: [your topic]
Data/content: [key facts and information]

Use the storytelling arc:
1. **The Setup** — Establish the current world (status quo)
2. **The Tension** — Introduce the problem or challenge
3. **The Journey** — What was tried, what we discovered
4. **The Revelation** — The key insight or turning point
5. **The Resolution** — The solution or recommendation
6. **The New World** — What changes if we act

For each act: slide title, key message, emotional tone, visual suggestion.
```

## Specific Presentation Types

### 8. Product Launch
```
Create a product launch presentation for [product name]:

What it is: [describe product]
Target audience: [who it's for]
Launch date: [when]
Key differentiator: [what makes it special]

Slides:
1. Teaser (build anticipation — question or provocative stat)
2. The problem (customer pain point, with empathy)
3. Existing solutions and their limitations
4. [Product name] introduction (the reveal moment)
5. Key features (3 features, each with a visual)
6. How it works (3-step process, simple)
7. Customer benefit / ROI
8. Pricing and plans
9. Launch offer / early access
10. Next steps / CTA

Design notes: suggest transitions for the reveal moment.
```

### 9. Case Study Presentation
```
Create a case study presentation:

Client: [company name or anonymized]
Challenge: [what problem they had]
Solution: [what you provided]
Results: [metrics, outcomes]

Slides:
1. Title: "[Client] achieved [result] with [your product/service]"
2. Client overview (industry, size, context)
3. The challenge (specific, quantified problems)
4. Why existing solutions didn't work
5. Our approach (methodology, timeline)
6. Implementation highlights (key moments)
7. Results (before vs after, with metrics)
8. Client quote / testimonial
9. Lessons learned
10. How this applies to the audience

Make the client the hero of the story, not your product.
```

### 10. Academic Research Presentation
```
Create a research presentation for [conference/defense]:

Paper title: [title]
Research question: [question]
Methodology: [approach]
Key finding: [main result]

Slides:
1. Title slide (title, authors, institution, date)
2. Motivation (why this research matters)
3. Research question and hypothesis
4. Literature review (key prior work, gap identified)
5. Methodology (with diagram)
6. Data description (source, size, preprocessing)
7. Results — main findings (key chart/table)
8. Results — supporting findings
9. Discussion (interpretation, implications)
10. Limitations and future work
11. Conclusion (restate contribution)
12. References (key citations)
13. Thank you + Q&A

Time: [X minutes]. Include timing per slide.
```
